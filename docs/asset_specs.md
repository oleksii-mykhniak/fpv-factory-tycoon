# Asset Spec Sheets — Стадія 1

*Технічні завдання на 3D-моделі й текстури. Кожна модель експортується у `.glb` і кладеться в `src/assets/models/`.*
*Розміри звірені з процедурними примітивами у `src/scene/scene.js` — моделі мають ставати в ті самі світові координати (fallback ≈ .glb).*
*Створено 2026-06-20.*

> **Як читати:** спершу §0 (спільний стиль — обов'язково для єдиного вигляду), далі spec кожної моделі. «Габарити» — у метрах, як у Babylon-сцені. «Origin» — куди ставиться нульова точка (визначає, як модель сідає у світ). «Named-вузли» — імена, за якими код адресує під-меші. «Якорі» — Empty-вузли для прив'язки ефектів/інших об'єктів.

## 0. Спільний style-guide (читати першим)

Усі моделі — свої й запозичені CC0 — мають виглядати з одного набору. Це досягається **спільною палітрою** і однаковими правилами.

### Палітра (узгоджена з поточною сценою)

| Роль | Hex | Де вживається |
|---|---|---|
| Тло сцени | `#121220` | clearColor (не модель) |
| Підлога | `#1e1e2a` | room |
| Стіни | `#252535` | room |
| Темний акцент / дверний отвір | `#1a1a28` | room, тіні |
| Дерево столу (світле) | `#6b4226` | workbench top |
| Дерево столу (темне) | `#5a3620` | workbench legs |
| Картон коробки | `#c49a3c` | delivery_box |
| Корпус дрона (темний) | `#2a2a3e` | drone body |
| Рама дрона (середній) | `#333355` | drone arms |
| Метал / мотори | `#444466` | nacelles, метал інструментів |
| Метал світлий / жало | `#5a5a7a` | iron tip, гвинти |
| Акцент-emissive (LED, нагрів) | `#e08a3c` | iron tip glow, LED дрона |
| Зелений LED / OK | `#7de07d` | індикатори |

> Запозичені CC0-моделі перефарбовуємо матеріалами під ці кольори, щоб не вибивались.

### Технічні правила (однакові для всіх)
- **Стиль:** low-poly, flat-shaded або з легким AO. Без glossy/metallic PBR (perf у WebView).
- **Матеріали:** один **palette-атлас** на всю гру (див. §6) АБО плоскі `StandardMaterial` за hex вище. Мета — мінімум draw calls.
- **Полігони:** бюджети нижче. Загалом — чим менше, тим краще; деталь читається силуетом і кольором, не геометрією.
- **Масштаб:** 1 одиниця Blender = 1 метр. Перед експортом — `Object ▸ Apply ▸ All Transforms` (scale = 1, rotation = 0).
- **Орієнтація:** моделюємо у Blender як є (Z-вгору); glTF-експортер з опцією **+Y Up** (дефолт) сам конвертує у Babylon Y-up.
- **Нормалі:** `Recalculate Outside` (Shift+N) перед експортом.

### Налаштування експорту glTF (однакові)
`File ▸ Export ▸ glTF 2.0 (.glb)`:
- Format: **glTF Binary (.glb)**
- Include: **Selected Objects** (експортуємо лише потрібний об'єкт + його якорі)
- Transform: **+Y Up** ✅
- Geometry: Apply Modifiers ✅, Normals ✅, UVs ✅ (Tangents ❌ — не треба без normal-map)
- Mesh: вершинні кольори ✅ якщо використовуємо vertex colors
- Animation: вимкнути для статичних моделей; для анімованих (коробка, гвинти) — увімкнути

## 1. `mini_drone.glb` — міні-дрон (X-frame)

Центральний об'єкт. Лежить на столі у фазах ASSEMBLY/READY. Має бути впізнаваним.

| Параметр | Значення |
|---|---|
| **Габарити** | діагональ рами ~1.62 м (розмах по кінцях плечей), корпус 0.88 × 0.88 м, висота ~0.18 м |
| **Origin** | центр X/Z, низ моделі (найнижча точка нацел) у Z=0 — щоб сідав на стіл |
| **Бюджет** | 600–1200 трикутників |
| **Світ. позиція (loader)** | ставиться на `anchor` дрона: `(0, 1.675, 0.15)` (= TABLE_TOP_Y + 0.1) |

### Named-вузли (parent = порожній `mini_drone` root)
- `body` — пласка центральна пластина 0.88 × 0.88 × 0.10, колір `#2a2a3e`
- `arm_x1`, `arm_x2` — два бруски 1.62 × 0.10 × 0.07, повернуті на ±45° (утворюють X), колір `#333355`
- `motor_FL`, `motor_FR`, `motor_RL`, `motor_RR` — мотори-нацели 0.22 × 0.22 × 0.14 у 4 кутах на ±0.48, колір `#444466`
- `prop_FL..RR` *(опц.)* — гвинти-диски над моторами, окремі об'єкти для анімації обертання (тест-польоту)
- `led_status` *(опц.)* — маленький куб з emissive `#7de07d`/`#e08a3c`

### Якорі (Empty, type Plain Axes, розмір 0.05)
- `anchor_solder_1` → біля `motor_FL` (де сипляться іскри пайки точки 1)
- `anchor_solder_2` → біля `motor_FR`
- `anchor_solder_3` → біля `motor_RL`
- `anchor_solder_4` → центр `body` (прошивка FC)
- `anchor_root` → origin дрона (точка постановки)

> 4 якорі = 4 точки пайки кіта `mini_drone` (`solderPointCount: 4`). Іскри/підсвітка під час міні-гри прив'язуються до `anchor_solder_{N}`.

### Blender — покроково
1. **Корпус:** Add ▸ Mesh ▸ Cube. S→0.44, 0.44, 0.05 (даст 0.88×0.88×0.10). Назви `body`. Матеріал `#2a2a3e`.
2. **Плечі:** Cube → S 0.81, 0.05, 0.035 → `arm_x1`. Rotate Z 45°. Дублювати (Shift+D) → `arm_x2`, rotate Z −90° (стане −45°). Матеріал `#333355`.
3. **Мотори:** Cube → S 0.11, 0.11, 0.07 → перемістити в (−0.48, −0.48, 0.04). Назви `motor_RL`. Дублювати в 3 інші кути → `motor_RR/FL/FR`. Матеріал `#444466`. *(Опц.)* зверху циліндр-`prop` (6–8 сторін) для анімації.
4. **Якорі:** Add ▸ Empty ▸ Plain Axes у кожну точку (див. вище). Перейменувати.
5. **Origin:** виділити всі меші ▸ Object ▸ Set Origin ▸ Origin to Geometry; потім підняти курсор у низ моделі і Set Origin ▸ Origin to 3D Cursor, щоб Z=0 був на дні. Центрувати по X/Y.
6. **Parent:** Add ▸ Empty (root) у origin, назвати `mini_drone`; виділити всі меші+якорі, Shift-клік root, Ctrl+P ▸ Object.
7. Apply All Transforms ▸ Recalculate Normals ▸ Export `mini_drone.glb`.

## 6. Текстури / матеріали

Стиль flat low-poly → текстур майже не треба. Два підходи (обрати один на всю гру):

### A. Palette-атлас (рекомендовано для perf)
- Одна маленька PNG `palette.png` (напр. 64×64) — сітка квадратів-кольорів з §0.
- Усі меші мають **один спільний матеріал** з цією текстурою; UV кожної грані вказує в потрібний квадрат.
- Результат: **1 draw call** на більшість статики (ціль GDD §9), єдиний стиль автоматично.
- Робота в Blender: розкласти UV граней у відповідні клітинки атласу.

### B. Плоскі StandardMaterial (простіше, більше draw calls)
- Кожен колір — окремий `StandardMaterial` за hex (як зараз у `scene.js`).
- Простіше робити, але більше матеріалів → більше draw calls. Ок для Стадії 1, гірше масштабується на фабрику.

### Опційно: baked AO
- Запекти ambient occlusion у вершинні кольори (vertex colors) або в текстуру — дешева об'ємність без динамічного світла. Узгоджується з «baked lighting» з GDD §9.
- Не PBR, не normal maps — це вб'є perf у WebView.

### Текстурні файли під Стадію 1
| Файл | Призначення | Розмір |
|---|---|---|
| `palette.png` | спільний атлас кольорів (підхід A) | 64×64 |
| `box_label.png` *(опц.)* | наклейка на коробці | 128×128 |
| `ao_*.png` *(опц.)* | запечене AO на модель | 256–512 |
